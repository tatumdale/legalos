# SKILL.md - Vendor Agreement Status

## name
vendor-agreement-status

## description
Check the status of existing agreements with a vendor across all connected systems. Provides a consolidated view of the legal relationship.

## Vendor Agreement Status Skill
You check the status of existing agreements with a vendor across all connected systems.

**Important:** You do not provide legal advice. Agreement status reports should be verified against original documents.

## Workflow

### Step 1: Identify the Vendor
Handle: full legal name vs trade name, abbreviations, parent/subsidiary relationships. Ask to clarify if ambiguous.

### Step 2: Search Connected Systems
In priority order: CLM (all contracts, active/expired/in negotiation), CRM (account status, contacts), Email (6 months of contract-related correspondence), Documents (executed agreements, drafts, due diligence), Chat (3 months of relevant mentions).

### Step 3: Compile Agreement Status
For each agreement: name and reference, status, key dates (effective, expiration, renewal notice deadline), key terms summary, outstanding items.

### Step 4: Gap Analysis
Identify missing agreements. Flag gaps based on relationship type (e.g., MSA without DPA when vendor handles personal data).

### Step 5: Generate Report
Report template covers: account overview, each agreement with status/dates/terms/outstanding items, gap analysis, key contacts, upcoming deadlines, recommended actions.

## Notes
- If no agreements found, report clearly
- For vendor groups, ask whether to check specific entity or entire group
- Flag expired agreements with surviving obligations (confidentiality, indemnification)
- Highlight agreements approaching expiration (within 90 days)
